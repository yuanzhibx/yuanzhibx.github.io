通用页面跳转实现

页面分析

我们观察 index.jsp，发现如果想要跳转页面，需要页面中发起ajax请求。之后在Controller中需要接收改请求

IndexController

需求：使用一个方法实现网页跳转

RestFul风格①

  1. 语法:
    1. 使用 / 方式分割参数
    2. 使用 {} 包裹参数
    3. 在参数方法中 动态接收参数时使用特定注解@PathVariable
  2. 注解: @PathVariable
    • name / value 表示接收参数的名称
    • required
  3. 作用
    1. 动态获取 url 路径中的参数
    2. 以统一的 url 地址, 不同请求类型, 实现不同业务的调用

RestFul风格②

  1. 思路: 利用通用的url地址,实现不同的业务调用
路径 操作 请求类型
www.jt.com/user?id=1 查询 GET
www.jt.com/user?id=1 删除 DELETE
www.jt.com/user?id=1&name=xxx 更新 PUT
www.jt.com/user 新增 POST
  1. 注解实现

总结两种用法

  1. 动态获取url路径中的参数
  2. 以统一的url地址, 不同的请求类型, 实现不同业务的调用一般都会添加请求类型进行标识, 为了安全

全局异常处理机制

分析

在如上商品新增操作中使用了 try-catch 语句来处理异常, 但如果代码中频繁出现 try-catch, 则可能影响代码结构.不便于阅读。 所以我们应该利用全局的异常的捕获机制,简化 try-catch 个数。

捕获位置: 常规捕获的位置是 Controller 层，因为 Controller 层是业务调用的最后的控制层。
