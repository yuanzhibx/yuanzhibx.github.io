第1章 绪论

1.1.1 数据库系统的基本概念

  • 真题
    • 【2019 选择】DBS 包括哪些组成部分？
    • 【2020 选择】DBS 包括哪些组成部分？
    • 【2021 选择】DB、DBMS、DBS 的包含关系？
  • 答案
    • 数据库系统（DBS） = 数据库（DB） + 数据库管理系统（DBMS） + 应用系统 + 数据库管理员（DBA）。

1.1.2 数据库的特点

  • 真题
    • 【2019 简答】简述数据库的基本特征。
    • 【2022 简答】简述数据库的基本特征。
  • 答案
    • 数据按模型组织、存储，可多用户共享，冗余度小，数据独立性高，由 DBMS 统一管理。

1.1.3 数据库管理系统的功能

  • 真题
    • 【2019 简答】数据库管理系统的功能有哪些？
    • 【2023 简答】简述数据库的事务管理功能。
  • 答案
    • DBMS 的功能：
      1. 数据定义
      2. 数据组织、存储与管理
      3. 数据操纵
      4. 事务管理与运行管理（含并发控制、恢复、安全性、完整性）
      5. 数据库的建立与维护
      6. 其他功能（字典、性能监视等）
    • 事务管理功能：保证事务的ACID特性（原子性、一致性、隔离性、持久性）。

第2章 关系模型

2.1.2 关系模式 / 2.1.3 关系数据库

  • 真题
    • 【2020 简答】什么是主码、外码、候选码？
    • 【2025 选择】主码与外码的区别？
  • 答案
    • 候选码：能唯一标识元组的属性或属性组合。
    • 主码：从候选码中选出的一个。
    • 外码：不是自身主码，但引用另一关系的主码。
    • 区别：主码唯一标识本关系元组；外码用来建立表间联系。

2.3 关系的完整性

  • 真题
    • 【2021 简答】关系的约束包括哪三类？实体完整性是什么？
  • 答案
    • 三类约束：实体完整性、参照完整性、用户定义完整性。
    • 实体完整性：主码属性不能为空（主码值不可为 NULL）。

2.4 关系代数

  • 真题
    • 【2019 选择】关系代数的几个符号有哪些？
    • 【2019 简答】关系代数的基本操作有哪些？
    • 【2023 简答】简述关系代数的基本操作。
  • 答案
    • 常用符号：选择（σ）、投影（π）、并（∪）、差（－）、笛卡尔积（×）、连接（⋈）、除（÷）。
    • 基本操作：选择、投影、并、差、笛卡尔积。
    • 专用操作：连接、除等。

第3章 SQL

3.2 数据定义（DDL）

  • 真题
    • 【2020 综合题】修改 student 表中学号字段为 int。
    • 【2021 综合题】建立课程表 course，课程号为主键。
    • 【2022 SQL题】创建学生表（含主码）。
    • 【2023 SQL题】写建表语句：顾客表、商品表、订购表。
    • 【2025 SQL题】创建课程表（含外码约束）。
  • 答案
    • 修改字段
      ALTER TABLE student ALTER COLUMN 学号 INT;
    • 建课程表
      CREATE TABLE course (课程号 CHAR(4) PRIMARY KEY, 课程名 CHAR(40) NOT NULL, 先修课 CHAR(4), 学分 SMALLINT, FOREIGN KEY (先修课) REFERENCES course(课程号));
    • 学生表
      CREATE TABLE student (学号 CHAR(10) PRIMARY KEY, 姓名 CHAR(20), 性别 CHAR(2), 年龄 INT);
    • 顾客/商品/订购表
      CREATE TABLE customer (顾客号 CHAR(6) PRIMARY KEY, 姓名 CHAR(20), 性别 CHAR(2));
      CREATE TABLE product (商品号 CHAR(6) PRIMARY KEY, 商品名 CHAR(40), 单价 DECIMAL(8,2));
      CREATE TABLE orderinfo (顾客号 CHAR(6), 商品号 CHAR(6), 数量 INT, PRIMARY KEY(顾客号, 商品号), FOREIGN KEY (顾客号) REFERENCES customer(顾客号), FOREIGN KEY (商品号) REFERENCES product(商品号));
    • 课程表（外码约束）
      CREATE TABLE course (课程号 CHAR(4) PRIMARY KEY, 课程名 CHAR(40), 学分 SMALLINT, 先修课 CHAR(4), FOREIGN KEY (先修课) REFERENCES course(课程号));

3.3 数据查询（DQL）

  • 真题
    • 【2020 综合题】解释 SQL 查询语句的含义。
    • 【2021 综合题】解释 SQL 定义视图语句的含义。
    • 【2023 SQL题】查询女性用户订购的商品信息。
    • 【2025 SQL题】SQL 查询语句（解释含义）。
  • 答案
    • 查询语句基本形式
      SELECT <列名> FROM <表或视图> WHERE <条件>;
    • 查询女性用户订购商品
      SELECT c.顾客号, c.姓名, p.商品号, p.商品名, o.数量 FROM customer c, product p, orderinfo o WHERE c.性别='女' AND c.顾客号=o.顾客号 AND p.商品号=o.商品号;
    • 定义视图的查询语句：生成虚拟表，简化查询、提高安全性。

3.4 数据更新（DML）

  • 真题
    • 【2019 SQL题】写 SQL 更新语句。
    • 【2020 综合题】写 SQL 删除语句。
    • 【2021 综合题】写 SQL 修改语句。
    • 【2022 SQL题】写 SQL 更新语句。
    • 【2025 SQL题】写 SQL 更新语句。
  • 答案
    • 更新： UPDATE student SET 系='计算机系' WHERE 姓名='张三';
    • 删除： DELETE FROM student WHERE 年龄 < 18;
    • 修改： UPDATE student SET 部门='研发部' WHERE 工号='1001';

3.6 视图

  • 真题
    • 【2019 SQL题】写创建视图语句。
    • 【2021 综合题】SQL 定义视图语句：解释含义。
    • 【2022 SQL题】建立视图查询计算机系学生。
    • 【2023 SQL题】创建视图 gm_view 查询顾客号、顾客名、订购的商品号和数量。
  • 答案
    • 计算机系学生视图
      CREATE VIEW cs_student AS SELECT 学号, 姓名 FROM student WHERE 系='计算机系';
    • 视图 gm_view
      CREATE VIEW gm_view AS SELECT c.顾客号, c.姓名, o.商品号, o.数量 FROM customer c, orderinfo o WHERE c.顾客号=o.顾客号;
    • 视图是虚拟表，不存储数据，只保存定义。

第6章 关系数据理论

6.2 范式 / 模式分解

  • 真题
    • 【2019 简答】简述基本表和视图的联系与区别。
    • 【2021 简答】基本表和视图的联系与区别。
  • 答案
    • 基本表：实际存在的表，独立存储数据。
    • 视图：虚拟表，不存储数据，仅保存定义。
    • 联系：视图派生自基本表，操作视图会影响基本表。
    • 区别：基本表是物理存储；视图是逻辑显示。
