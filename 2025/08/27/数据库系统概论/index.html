数据库系统概论（第六版）

第一章 绪论

1.1 数据库系统概述

1.1.1 数据库四个基本概念 （【2019、2020 选择】DBS 组成；【2021 选择】DB/DBMS/DBS 关系）

  • 数据：描述事物的符号记录。
  • 数据库：长期存储在计算机内有组织、可共享的大量数据的集合。
  • 数据库管理系统：
    • 功能：
      1. 数据定义功能
      2. 数据组织、存储和管理功能
      3. 数据操纵功能
      4. 数据库的事务管理和运行管理功能
      5. 数据库的建立和维护功能
      6. 其他功能
  • 数据库系统：指由数据库、数据库管理系统、应用系统、数据库管理员组成的整体。

1.1.2 数据管理技术三阶段

  1. 人工管理
    • 特点：
      • 数据不保存
      • 应用程序管理数据
      • 数据不共享
      • 数据不具有独立性（数据独立性是指数据与应用程序相互独立）
  2. 文件系统
    • 特点：
      • 数据可长期保存
      • 数据由专门的软件进行管理
    • 缺点：
      • 数据共享性弱，冗余度高
      • 数据独立性弱
  3. 数据库系统 （【2019、2022 简答】数据库的基本特征）
    • 特点：
      • 整体数据的结构化
      • 数据的共享性强、冗余度低且易于扩充
      • 数据的独立性强（包括数据的物理独立性和数据的逻辑独立性）
      • 数据由数据库管理系统统一管理和控制

1.1.4 数据库管理系统的功能 （【2019 简答】DBMS 功能；【2023 简答】事务管理功能）

  1. 数据定义功能
  2. 数据组织、存储和管理功能
  3. 数据操纵功能
  4. 数据库的事务管理和运行管理功能
  5. 数据库的建立和维护功能
  6. 其他功能

1.2 数据模型

1.2.2 概念模型 （【2020、2022、2023 选择】E-R 图模型）

信息世界中的基本概念：

  • 实体：客观存在并可相互区别的事物
  • 属性：实体所具有的某一特性
  • 码：唯一标识实体的属性集
  • 实体类型：具有相同属性的实体必然具有共同的特征和性质
  • 实体集：同一类型实体的集合
  • 联系：事物内部以及事物之间是有联系的
    • 实体内部的联系：指组成实体的各属性之间的联系
    • 实体之间的联系：指不同实体集之间的联系

1.2.3 数据模型的三要素 （【2020 简答】【2022 选择】【2023 简答】【2025 简答】数据模型的三要素）

  • 数据结构：描述数据库的组成对象以及对象之间的联系。
  • 数据操纵：指对数据库中各种对象（型）的实例（值）允许执行的操作的集合。
  • 完整性约束：一组完整性约束（逻辑上的）。

1.2.4 常见数据模型

  1. 层次模型
    • 条件：
      • 有且只有一个节点没有双亲结点，这个节点称为根结点
      • 根以外的其他结点有且只有一个双亲结点
    • 优点：
      • 层次模型的数据结构比较简单清晰
      • 层次数据库的查询效率高
      • 层次数据模型提供了良好的完整性约束
    • 缺点：（文档中未提及，此处留白）
  2. 网状模型
    • 条件：
      • 允许一个以上的节点无双亲结点
      • 一个结点可以有多于一个的双亲结点
    • 优点：
      • 能够更为直接的描述现实世界
      • 具有良好的性能，存取效率较高
    • 缺点：（文档中未提及，此处留白）
  3. 关系模型
    • 术语：
      • 关系
      • 元组
      • 属性
      • 码
      • 域
      • 分量
      • 关系模型
    • 优点：
      • 关系模型建立在严格的数学概念基础上
      • 关系模型的概念单*

第二章 关系模型

2.1 关系模型的数据结构及形式化定义

2.1.1 关系

  • 域：一组具有相同数据类型的值的集合。
  • 笛卡尔积：域上的一种集合运算。
  • 元组：关系中的每个元素。
  • 分量：元组中的每个值。
  • 关系：（文档中未完整定义，此处留白）

2.1.2 关系模式 （【2020 选择】主码外码区别；【2025 选择】主码外码区别）

  • 关系模式是 “型”，关系是 “值”。
  • 关系模式：关系的描述，是静态的、稳定的。
  • 关系：关系模式在某一时刻的状态或内容，是动态的，随时间不断变化的。
  • 候选码：关系模式中的某一个属性或一组属性的值能唯一的表示一个元组，而它的真子集不能唯一的表示一个元组。
  • 主码：从多个候选码中选定其中一个。
  • 主属性：候选码的诸属性。
  • 非主属性：不在任何候选码的属性。
  • 全码：关系模式中的所有属性是候选码。

2.2 关系操作

2.2.1 基本关系操作 （【2019 简答】关系代数操作；【2023 简答】关系代数操作）

  • 查询操作：选择、投影、连接、除法、并、差、交、笛卡尔积；其中基本操作包括选择、投影、并、差、笛卡尔积。
  • 更新操作：插入、删除、修改操作。

2.2.2 关系数据语言的分类

关系数据语言分为关系代数、关系演算、结构化查询语言 SQL 三类。

2.3 关系的完整性 （【2021 简答】关系完整性约束）

关系的完整性包括三类约束：

  • 实体完整性
  • 参照完整性
  • 用户定义完整性

2.4 关系代数 （【2019 选择】关系代数符号）

关系代数：是一种抽象的查询语言，它用对关系的运算来表达查询。其三大要素为运算对象、运算符、运算结果。

2.4.1 传统的集合运算

传统的集合运算包括并、差、交、笛卡尔积。

2.4.2 专门的关系运算

专门的关系运算包括选择、投影、连接、除。

3.3.3 嵌套查询

一个 SELECT-FROM-WHERE 语句称为一个查询块，将一个查询块嵌套在另一个查询块的 WHERE 子句或 HAVING 短语的条件中的查询称为嵌套查询，上层的查询块称为外层查询或父查询，下层查询块称为内层查询或子查询，SQL 语言允许多层嵌套查询，即一个子查询中还可以嵌套其他子查询，子查询不能使用 ORDER BY 子句。

  1. 带有 IN 谓词的子查询
    例：查询与 “刘晨” 在同一个系学习的学生。

  3. 带有 ANY 或 ALL 谓词的子查询
    使用 ANY 或 ALL 谓词时必须同时使用比较运算，语义如下表：

    谓词 语义
    >ANY 大于子查询结果中的某个值
    >ALL 大于子查询结果中的所有值
    <ANY 小于子查询结果中的某个值
    <ALL 小于子查询结果中的所有值
    >=ANY 大于等于子查询结果中的某个值
    >=ALL 大于等于子查询结果中的所有值
    <=ANY 小于等于子查询结果中的某个值
    <=ALL 小于等于子查询结果中的所有值
    =ANY 等于子查询结果中的某个值
    =ALL 等于子查询结果中的所有值
    !=ANY 不等于子查询结果中的某个值
    !=ALL 不等于子查询结果中的任何一个值

    带有 EXISTS 谓词的子查询不返回任何数据，只产生逻辑真值 “true” 或逻辑假值 “false”。

    • 若内层查询结果非空，则外层的 WHERE 子句返回真值。
    • 若内层查询结果为空，则外层的 WHERE 子句返回假值。
      由 EXISTS 引出的子查询，其目标列表达式通常都用 * ，因为带 EXISTS 的子查询只返回真值或假值，给出列名无实际意义。
      NOT EXISTS 谓词：
    • 若内层查询结果非空，则外层的 WHERE 子句返回假值。
    • 若内层查询结果为空，则外层的 WHERE 子句返回真值。
      例：查询这样一个同学，没有一门课是他不选修的。

3.6 视图 （【2021 综合题】解释 SQL 定义视图语句）

概念：视图是从一个或几个基本表（或视图）导出的表。它与基本表不同，是一个虚表，是一组数据的逻辑表示。其本质是对应与一条 select 结果集被赋予一个名字，即视图名字。数据库只存放视图的定义，而不存放视图对应的数据，这些数据仍存放在原来的基本表中，一旦基本表中的数据发生变化，从视图中查询出的数据也就随之改变。

作用（优点）：

  1. 视图能够简化用户的操作。

  2. 视图能够使用户以多种角度看待同一问题。

  3. 视图对重构数据库提供了一定程度的逻辑独立性。

  4. 视图能够对机密数据提供安全保护。

  5. 适当利用视图可以更清晰地表达查询。
